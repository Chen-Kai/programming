### 什么是数据交换
- 狐狸：考拉，把你的个人信息发给我
- 考拉：9，8，爬树

上例中，考拉和狐狸就在进行数据交换――考拉给出数据，狐狸理解数据。

### 为什么需要规定数据格式
虽然考拉给出了信息，可狐狸无法理解。*9*是什么？*8*和*爬树*又是什么？ 为了解决这个问题，就需要规定数据的格式。例如：

<pre>
    年龄:9,
    体重:8,
    爱好:爬树
</pre>

这下狐狸就明白了，`年龄:9`是一条信息。`年龄`是信息名称（键），`9`是信息内容（值），相当于`年龄是9岁`。每条信息都由`键:值`对组成，并由`,`分隔。

### JSON是什么
JSON定义了数据交换的格式，一条JSON数据就是一些`键:值`对的集合。

### JSON的键是什么
是且只能是字符串。例如："name"

### JSON的值是什么
JSON的值可以是：

- 字符串
    - 规则：以 \" 开始，以 \" 结尾，支持 \ 转义
    - 例子：  
		`"string"`  
		`"His favorite English word is \"courage\""`

- 数字
    - 规则：以数字或-（负号）开头，可以包含.（小数点）
    - 例子：  
        `2015`  
        `3.14`
- 对象
    - 规则：以{开始，以}结尾，中间包含一些键值对，以,分隔。
    - 例子：  
        `{"name":"koala", "age":9, "weight":8, "habit":"climbing"}`  
- 数组
    - 规则：以\[开头，以\]结尾，中间包含一些值，以,分隔
    - 例子：  
        `["Monday", "Tuesday", "Wednesday", "Thursday", "Friday", "Saturday", "Sunday"]`  
        `"array":[1, 10, 100]`
- true
    - 规则：内置的boolean类型
    - 例子：  
        `"boolean":true`
- false
    - 规则：内置的boolean类型
    - 例子：  
        `"boolean":false`
- null
    - 规则：表示空，未设置
    - 例子：  
        `"name":null`
